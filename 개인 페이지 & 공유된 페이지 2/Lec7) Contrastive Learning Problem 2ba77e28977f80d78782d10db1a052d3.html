기존 Self-Supervised Learning의 문제점

  • PreText-Task를 수행함으로써 → 데이터의 구조적인 특성을 이해하려고 함
    • example
      • 이미지 회전 시켜 놓고 → 몇 도 회전 했는가?
  • 이것이 이미지의 일반적인 특징을 학습시킬 수 있는가?
    • 회전 맞추기 → 사물의 위/아래 방향 정보에만 집중할 가능성 → Representation이 Pretext Task에 최적화될 가능성
    • ∴\therefore∴﻿ Pretext Task의 목표가, 고수준 시각적 표현 학습과 관련 없을 수 있음
    • 또한 PreText Task라는 인위적인 규칙이 → 일반적인 시각정보에 최적화 X이

Contrastive Learning이 해결하는 방향성

  • 같은 이미지의 증강표현 View는 → Representation 공간내에서 가깝게
  • 다른 이미지의 증강 표현 View는 → Representation 공간내에서 멀게
  • ∴\therefore∴﻿ 인위적인 PreText-Task 학습 X
    • 본질적으로 같은 이미지면 가까워야 한다, 다른 이미지면 멀어야 한다 학습

Contrastive Learning의 주요 이슈

  • 본질적으로 완벽하게 Contrastive Learning으로 High-level Representation을 얻기는 힘들다

문제1) Postive Pair 정의가 모호하다

  • 어디까지가 ‘같은 이미지’인가?에 대한 물음
  • Example
    • 이때 (x2,x3)(x_2, x_3)(x2​,x3​)﻿는 non-Postive임
    • 무조건, 같은 원본 이미지에서 나온 view만 Postive이다
    • 이미지 A와 B는 둘다 강아지 → semantic level에선 유사하다
    • 다만, Contrastive Learning은 서로 멀어지도록 학습해버림
    • 따라서, 강아지들을 서로 비슷하게 인식하는 능력 ↓\downarrow↓﻿
      • Semantic 구조를 자연스럽게 배우기 어려움

문제2) Augmentation이 Representation을 뒤흔든다

  • Augmentation이 랜덤하게 적용됨 → 같은 이미지인데도, 너무 달라보이게 변형시킬 수 있다
  • 모델은 원본 고유 특징 보다, Transformation 기준으로 판단 기준을 잡을 수 있다

본질적인 난제) Instance Discrimination

  • Contrastive Learning의 실제 목표 = 각 이미지의 Instance를 구분하는 능력
    • 같은 강아지라도 푸들, 요스셔테리어는 다르게 학습된다
  • But, 우리가 DownStream Task에서 원하는 것 → 같은 클래스의 공유된 구조 표현 and Semantic-level의 일반화된 패턴
  • ∴\therefore∴﻿ Contrastive Learning은 실제로 잘 동작하지만, 근본적으로 해결하기 어려운 문제점을 가지고 있다

Contrastive Learning의 주요 이슈: Solution

  • Instance Discrimination에 대한 완벽한 해결책 X
  • negative sample ↑\uparrow↑﻿ → 이미지 구분 능력 ↑\uparrow↑﻿ → 그와 동시에 semantic Mismatch 현상 ↑\uparrow↑﻿
    • Semantic Mismatch: 의미적으로 비슷한 것들도 억지로 멀이지게 만듦
    • Representation이 Instance 단위로 유사/비유사 경계를 정함
    • 따라서, negative 의존도를 줄이려고 함

  • Queue안에 있는 feature를 클러스터링 (Deep Cluster 이용)
    • Queue안에 있는 Instance-level의 Key를 Clustering 하여 Semantic한 Key로 바꾸자
    • example

      • semantic Mismatch 문제 완화 가능
  • 클러스터링 방식의 근본적 문제

      • 클러스터링 수행 시 → 같은 클러스터로 묶일 수 있다
      • 의미적으로는 다르지만, 같은 클러스터에 속하게 되면서 → 그 잘못된 grouping이 학습 과정에서 반복적으로 강화될 위험이 있다
      • example

        • 이 경우, 사람과 개를 구분하는 decision boundary가 제대로 형성되지 못할 수 있다

Prototypical Contrastive Learning

  • 기존: 각 Instance 별로 Postive와는 가까이, Negative와는 멀리
  • Prototype wise:
    • 표현 공간 상의 feature들을 클러스터링
    • 각 클러스터의 대푯값이랑 → Postive와는 가까이, negative와는 멀리 하는 방식임
    • 최종적으로는 기존 방식 + Prototype wise 방식 둘다 사용한다
  • 이에 Prototypical 방식은 semantic Structure도 유지하고, 기존 근본적 문제인 개별 이미지간의 semantic한 차이도 일부 반영 가능

1) Prototypical Contrastive Learning Algorithm

  • EM 알고리즘을 사용하여 최적화한다 (E-step → M-step → E-step → ⋯\cdots⋯﻿ → M-step)
  • E-step
    • Momentum Encoder로 전체 이미지 통과 → Feature 얻음
    • K-means로 모든 feature들을 클러스터링함
    • 각 Prototype(클러스터의 대푯값) 주변의 분포 계산
      • 해당 centroid 주변으로 feature들이 퍼져있는지 좁게 있는지를 알기 위해서
      • Example
        • A Cluster: 넓게 퍼짐 → 해당 클러스터내 Instance는 Semantic 적으로 일관성 ↓\downarrow↓﻿
        • B Cluster: 오밀조밀함 → Semantic 적으로 일관성 ↑\uparrow↑﻿
  • M-step
    • E-step에서 만든 Representation space내 Cluster를 바탕으로 Encoder를 학습시킨다
    • 이때 ProtoNCE Loss를 계산한다
      • Instance wise-Loss와 Prototype-wise Loss가 같이 있음

2) ProtoNCE Loss

  • 기존의 Instance 방식과 Prototype 방식을 둘다 사용하게 됨
  • (1) Instance Contrastive Loss
    • 기존과 동일함
    • 각 viv_ivi​﻿ = i번째 view의 Representation feature
    • Postive pair feature → (vi,vj)(v_i, v_j)(vi​,vj​)﻿ ⇒ 이 둘은 서로 가깝게
    • 개별 이미지 거리를 구분하도록 학습됨
  • (2) Prototype Contrastive Loss
    • 수식 해석
      • CsmC_s^mCsm​﻿: Postive Pair인 VjV_jVj​﻿가 속한 Cluster의 centroid
      • ϕsm\phi_s^mϕsm​﻿: 해당 클러스터가 얼마나 퍼져 있는가?
    • viv_ivi​﻿가 vjv_jvj​﻿가 속한 클러스터의 centroid와 가까워지도록 함
    • viv_ivi​﻿가 negative에 해당하는 클러스터와는 멀어지도록 함
    • ϕ\phiϕ﻿의 역할 = 기존의 temperature 역할 수행

3) 해당 Concept에서 Semantic Meaning을 더 잘할 수 있는 방법

  • (1) Instance-level의 한계
    • Instance NCE는 각 이미지를 독립된 instance로 취급함
      • 같은 개(요크셔, 푸들)도 → 서로 다른 instance이므로 negative로 처리됨
      • 사람과 개 → 둘 다 똑같이 negative로 간주
      • ∴\therefore∴﻿ semantic similarity가 loss에 반영 X
    • 결과적으로 같은 semantic 내부에서도 과도하게 분리 → feature 공간이 semantic하게 조직되지 않음
  • (2) PCL의 핵심: Instance + Prototype 이중 구조
    • 기존 Instance loss + Prototype Contrastive Loss 추가
    • 모델이 받는 두 가지 압력
      • Instance Contrastive Loss
        • 같은 이미지의 augmentation끼리 가깝게
        • 다른 instance는 멀어짐
      • Prototype Contrastive Loss
        • positive pair가 속한 cluster 중심(semantic group)과 가까워지도록 feature 조정
        • semantic group에 맞게 압축하는 힘이 작용
    • ∴\therefore∴﻿ 같은 종(개)의 다양한 형태(요크셔, 푸들)도 → 같은 semantic cluster로 모여듦
  • (3) 결과: Semantic Meaning 강화
    • 같은 개 종류끼리 → 생김새가 달라도 같은 cluster 중심으로 끌림
    • 개와 사람처럼 의미적으로 다른 것 → prototype level에서 더 강하게 분리
    • ∴\therefore∴﻿ Instance NCE의 미시적 구분 + Prototype NCE의 거시적 의미 구조 = 더 강한 semantic representation

Bootstrap Your Own Latent - Concept

  • Step 1: 모델 A를 라벨 데이터로 사전 학습 (backbone 고정, head만 학습)
    • 이 단계에서 성능은 매우 낮음
    • 하지만 이 과정이 없으면 → 모델 A의 출력이 완전한 noise가 됨
    • ∴\therefore∴﻿ Model A + Linear가 입력에 대해 최소한의 일관성을 확보하는 중요한 역할
  • Step 2: 라벨 없는 데이터(새로운 데이터)에서 모델 A의 출력을 pseudo-label로 사용
  • Step 3: 해당 pseudo-label로 모델 B를 Supervised Learning 방식으로 학습
    • 놀랍게도 실제 성능이 향상됨
  • 핵심: 학습되지 않은 모델 A가 생성한 pseudo-label이 → 모델 B의 학습에 실제로 도움이 되는 현상 발생

1) 왜 성능 향상?

  • 추가된 정보량이 없는데, Pseudo-label을 통한 학습이 왜 성능이 향상되었는가?
  • A모델 + Linear가 데이터 전체에 대해 일관된 패턴을 제공 → B모델이 이를 바탕으로 Representation 구조를 회복
  • 정보가 정렬된 것
    • Step1에서 A는 랜덤이지만, 같은 입력 X에 대해 같은 출력 → 일관성 확보
    • 비슷한 이미지 입력 → 비슷한 출력을 낼 가능성 ↑\uparrow↑﻿
    • 완전히 다른 이미지 입력 → 다른 출력을 낼 가능성 ↑\uparrow↑﻿
    • 실제 데이터가 무엇인지는 모르지만 → 이들 간의 상대적 거리는 부분적으로 보존됨
    • ∴\therefore∴﻿ B모델은 정확한 라벨 없이도 같은 Pseudo-label을 가진 이미지들을 Representation에서 가까이 배치
    • 결과: Representation geometry를 지속적으로 정렬하는 효과
